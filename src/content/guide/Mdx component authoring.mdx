MDX memungkinkan penulisan konten Markdown yang diselingi komponen interaktif. Di Astro, MDX component adalah file `.astro` biasa yang di-import dan digunakan langsung di dalam konten.

## Setup MDX di Astro

Pastikan integration MDX sudah terpasang:

:::code
```bash
npx astro add mdx
```

Di `astro.config.ts`:

```ts
import { defineConfig } from "astro/config";
import mdx from "@astrojs/mdx";

export default defineConfig({
  integrations: [mdx()],
});
```
:::

## Pola Komponen

### Props + slot

Pola paling umum — props untuk konfigurasi, slot untuk konten:

:::code
```astro
---
// src/components/mdx/Card.astro
interface Props {
  title:    string;
  variant?: "info" | "warning" | "success" | "error";
}
const { title, variant = "info" } = Astro.props;
---
<div class={`card card--${variant}`}>
  <p class="card__title">{title}</p>
  <div class="card__body"><slot /></div>
</div>
```
:::

Pemakaian di MDX:

:::code
```mdx
<Card title="Heads up" variant="warning">
  Perubahan ini bersifat breaking change di versi 2.0.
</Card>
```
:::

### Named slots

Untuk komponen yang butuh multiple content areas:

:::code
```astro
---
// Tabs.astro
const { tabs } = Astro.props;
---
<div class="tabs">
  {tabs.map((tab, i) => (
    <div class="tab-panel" data-index={i}>
      <slot name={tab.toLowerCase()} />
    </div>
  ))}
</div>
```
:::

## Tips Authoring

<Accordion title="Kenapa komponen MDX tidak perlu TypeScript?">
  Komponen `.astro` sudah di-type-check saat build. Props interface di frontmatter Astro cukup untuk keamanan tipe. TypeScript penuh hanya diperlukan untuk logika kompleks yang lebih baik dipisah ke file `.ts`.
</Accordion>

<Accordion title="Bagaimana pass Markdown ke dalam komponen?">
  Gunakan `<slot />` dan tulis konten sebagai Markdown biasa di antara tag komponen. Astro secara otomatis memproses Markdown di dalam slot MDX — heading, bold, link, semua berfungsi.
</Accordion>

<Accordion title="Bisakah komponen MDX punya client-side interactivity?">
  Ya, dengan menambahkan `client:load` atau `client:visible` directive. Namun untuk komponen docs seperti Tabs dan Accordion, vanilla JS di `<script>` tag lebih ringan dan tidak butuh hydration overhead.
</Accordion>

## Global MDX Components

Daripada import di setiap file, daftarkan komponen global di `astro.config.ts`:

:::code
```ts
import { defineConfig } from "astro/config";
import mdx from "@astrojs/mdx";

export default defineConfig({
  integrations: [
    mdx({
      components: {
        // Override default HTML elements
        pre:  "./src/components/mdx/CodeBlock.astro",
        img:  "./src/components/mdx/Image.astro",
        // Custom components tanpa import
        Card: "./src/components/mdx/Card.astro",
        Tabs: "./src/components/mdx/Tabs.astro",
      },
    }),
  ],
});
```
:::

Dengan ini, `<Card>` dan `<Tabs>` langsung bisa dipakai di semua MDX tanpa baris `import`.
