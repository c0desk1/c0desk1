Tailwind v4 memperkenalkan `@theme` directive — cara native untuk mendefinisikan design token langsung di CSS tanpa konfigurasi JavaScript. Ini mengubah cara kita membangun sistem desain.

## Mengapa Token System?

Design token adalah *single source of truth* untuk semua keputusan visual. Tanpa token, nilai seperti warna dan spasi tersebar di ratusan file — sulit di-maintain dan inconsistent.

:::info
Token yang terdefinisi dengan baik memungkinkan: rebrand dengan mengubah satu file, dark mode otomatis, dan konsistensi di seluruh komponen tanpa manual tracking.
:::

## Struktur `global.css`

:::code
```css
/* src/styles/global.css */
@import "tailwindcss";

@theme inline {
  /* Color scale */
  --color-void-0:    #ffffff;
  --color-void-950:  #0a0a0a;
  --color-void-1000: #000000;

  /* Accent */
  --color-accent: var(--color-void-400);

  /* Radius */
  --radius-sm:   4px;
  --radius:      6px;
  --radius-md:   8px;
  --radius-lg:   12px;
  --radius-full: 9999px;

  /* Easing */
  --ease-snappy: cubic-bezier(0.16, 1, 0.3, 1);
  --ease-out:    cubic-bezier(0, 0, 0.2, 1);

  /* Duration */
  --duration:    150ms;
  --duration-md: 250ms;
}
```
:::

### Semantic tokens di `:root`

:::code
```css
:root {
  --bg:        var(--color-void-0);
  --fg:        var(--color-void-900);
  --fg-muted:  var(--color-void-500);
  --border:    var(--color-void-200);
  --accent:    var(--color-void-400);
}

.dark {
  --bg:        var(--color-void-950);
  --fg:        var(--color-void-50);
  --fg-muted:  var(--color-void-300);
  --border:    var(--color-void-800);
}
```
:::

## Lapisan Token

Token sebaiknya dibagi tiga lapisan:

| Lapisan | Contoh | Keterangan |
|---------|--------|------------|
| **Primitive** | `--color-void-500` | Nilai mentah, tidak berubah per mode |
| **Semantic** | `--fg-muted` | Merujuk ke primitive, berubah per mode |
| **Component** | `--btn-bg` | Spesifik ke satu komponen |

## Custom Utilities

Dengan `@utility`, kamu bisa daftarkan utility class yang merujuk token:

:::code
```css
@utility btn-primary {
  background-color: var(--fg-strong);
  color: var(--bg);
  border: 1px solid transparent;

  &:hover {
    background-color: var(--fg-muted);
  }
}

@utility text-muted {
  color: var(--fg-muted);
}

@utility surface {
  background:    var(--bg-subtle);
  border:        1px solid var(--border);
  border-radius: var(--radius-lg);
}
```
:::

Lalu pakai di HTML:

```html
<button class="btn btn-primary px-4 py-2 text-sm">
  Deploy
</button>
```

## Dark Mode

:::code
```css
    /* global.css */
    @custom-variant dark (&:is(.dark *));

    .dark {
      --bg: var(--color-void-950);
      --fg: var(--color-void-50);
    }
```
:::

Toggle di JS:

:::code
```ts
    document.documentElement.classList.toggle("dark");
```
:::

:::code
 ```css
    @media (prefers-color-scheme: dark) {
      :root {
        --bg: var(--color-void-950);
        --fg: var(--color-void-50);
      }
    }
```
:::

Otomatis mengikuti OS, tapi tidak bisa di-override user.

## FAQ

<Accordion title="Apakah bisa mix Tailwind utilities dengan token?">
  Ya. Token CSS variable bekerja berdampingan dengan utility class Tailwind. Gunakan token untuk nilai yang perlu konsisten di tema, dan utility class untuk layout dan spacing yang bersifat satu kali pakai.
</Accordion>

<Accordion title="Apa bedanya @theme inline vs @theme biasa?">
  `@theme inline` membuat token *tidak* di-output sebagai class utility Tailwind — hanya tersedia sebagai CSS variable. Cocok untuk token internal yang tidak perlu di-expose ke HTML. Tanpa `inline`, Tailwind juga membuat class seperti `text-void-500`.
</Accordion>

<Accordion title="Bagaimana cara debug token yang tidak terapply?">
  Buka DevTools → Elements → Computed. Cari custom property yang dimaksud. Jika nilainya `initial`, berarti token belum terdefinisi di scope tersebut. Pastikan `:root` atau `.dark` class ada di ancestor element.
</Accordion>

## Kesimpulan

Token system yang baik adalah investasi jangka panjang. Dengan Tailwind v4 dan `@theme`, setup-nya lebih sederhana dan seluruhnya hidup di CSS — tidak perlu `tailwind.config.js` untuk nilai dasar.

Langkah selanjutnya: integrasikan token ini dengan komponen MDX dan pastikan semua komponen menggunakan `var(--token)` bukan hardcoded hex.
