Astro punya built-in i18n routing sejak v3.5 — tidak perlu `next-i18next`, `react-intl`, atau library berat lainnya. Panduan ini cover setup dari nol sampai production.

## Konfigurasi Dasar

:::code
```ts
// astro.config.ts
import { defineConfig } from "astro/config";

export default defineConfig({
  i18n: {
    defaultLocale: "id",
    locales: ["id", "en", "ru"],
    routing: {
      prefixDefaultLocale: false,
    },
  },
});
```
:::

:::info
`prefixDefaultLocale: false` berarti `/blog` → bahasa default (id), `/en/blog` → English, `/ru/blog` → Russian. Ini pattern yang paling SEO-friendly.
:::

## Struktur File

:::filetree
  - src/
    - pages/
      - blog/
        - **index.astro** ← default locale (id)
        - [slug].astro
      - [...locale]/
        - blog/
          - **index.astro** ← en, ru
          - [slug].astro
    - content/
      - blog/
        - **my-post.id.mdx** ← atau sub-folder per locale
        - my-post.en.mdx
    - i18n/
      - id.json
      - en.json
      - ru.json
:::

## Helper `useTranslations`

:::code
```ts
// src/i18n/index.ts
import id from "./id.json";
import en from "./en.json";
import ru from "./ru.json";

const translations = { id, en, ru } as const;
type Locale = keyof typeof translations;
type Keys   = keyof typeof id;

export function useTranslations(locale: Locale) {
  return function t(key: Keys): string {
    return translations[locale]?.[key] ?? translations["id"][key] ?? key;
  };
}

export function getLocaleFromUrl(url: URL): Locale {
  const [, first] = url.pathname.split("/");
  if (first in translations) return first as Locale;
  return "id";
}
```
:::

Pemakaian di komponen:

:::code
```astro
---
import { getLocaleFromUrl, useTranslations } from "@/i18n";

const locale = getLocaleFromUrl(Astro.url);
const t      = useTranslations(locale);
---

<nav>
  <a href={`/${locale === "id" ? "" : locale + "/"}blog`}>
    {t("nav.blog")}
  </a>
</nav>
```
:::

## Content per Locale
Naming convention dengan suffix locale:

:::filetree
- src/content/blog/
  - setup-astro.id.mdx
  - setup-astro.en.mdx
  - etup-astro.ru.mdx
:::

Di `content.config.ts`:

:::code
```ts
const blog = defineCollection({
  loader: glob({
    pattern: "**/*.{md,mdx}",
    base: "./src/content/blog"
  }),
  schema: z.object({
    lang: z.enum(["id", "en", "ru"]).default("id"),
    // ... field lainnya
  }),
});
```
:::

Subfolder per locale:

:::filetree
- src/content/blog/
  - id/
    - setup-astro.mdx
  - en/
    - setup-astro.mdx
  - ru/
    - setup-astro.mdx
:::

Query berdasarkan folder:

:::code
 ```ts
const posts = await getCollection("blog",
  ({ id }) => id.startsWith(`${locale}/`)
);
```
:::

## Fallback Content

Untuk artikel yang belum diterjemahkan, Astro bisa fallback ke locale default:

:::code
```ts
// astro.config.ts
i18n: {
  defaultLocale: "id",
  locales: ["id", "en", "ru"],
  fallback: {
    en: "id",
    ru: "id",
  },
}
```
:::

:::warning
Konfigurasi `fallback` bisa conflict dengan `[...locale]` catch-all route. Jika terjadi, hapus `fallback` dari config dan implementasikan manual di getStaticPaths.
:::

## Hreflang untuk SEO

Tambahkan di `<head>` untuk setiap halaman:

:::code
```astro
---
const supportedLocales = ["id", "en", "ru"];
const currentPath = Astro.url.pathname
  .replace(/^\/(en|ru)/, "")
  .replace(/\/$/, "") || "/";
---

{supportedLocales.map(locale => {
  const href = locale === "id"
    ? `${SITE.url}${currentPath}`
    : `${SITE.url}/${locale}${currentPath}`;
  return <link rel="alternate" hreflang={locale} {href} />;
})}
<link
  rel="alternate"
  hreflang="x-default"
  href={`${SITE.url}${currentPath}`}
/>
```
:::

## FAQ

<Accordion title="Apakah perlu library i18n eksternal?">
  Untuk kebanyakan blog, tidak. JSON + helper function sudah cukup. Library seperti `astro-i18n-aut` berguna jika kamu butuh pluralization kompleks, date formatting per locale, atau integrasi dengan CMS yang punya multi-language support.
</Accordion>

<Accordion title="Bagaimana handle URL slug yang berbeda per bahasa?">
  Buat lookup table di konten atau simpan `slug` per locale di frontmatter. Contoh: `slug.id: "setup-astro"`, `slug.en: "astro-setup"`. Lalu di getStaticPaths, generate path dari slug yang sesuai locale.
</Accordion>

<Accordion title="Sitemap multi-bahasa bagaimana?">
  Install `@astrojs/sitemap` dan tambahkan `i18n` config di sitemap integration. Secara otomatis akan generate sitemap dengan hreflang annotations per URL.
</Accordion>

## Kesimpulan

i18n di Astro jauh lebih simple dari yang dibayangkan. Routing sudah built-in, content bisa diorganisir dengan naming convention sederhana, dan SEO multi-bahasa bisa diimplementasi tanpa plugin tambahan.

Yang paling penting: mulai dengan satu bahasa tambahan saja. Menambah bahasa ketiga jauh lebih mudah setelah sistem pertama berjalan.
